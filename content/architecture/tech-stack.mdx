# Tech Stack

## Rationale

- **Next.js** — Full-stack React, great DX, Vercel deployment
- **Node.js/Express** — Backend team familiarity, rich ecosystem
- **PostgreSQL** — Relational data, ACID, mature tooling

## Alternatives Considered

- Remix vs Next.js — Chose Next for ecosystem and deployment
- Go vs Node — Chose Node for team velocity and shared language with frontend
